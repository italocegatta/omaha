# 1.2 PATCH /api/assets/{asset_id} — invalid sum 404

## Reproduction
```bash
uv run pytest tests/test_t99_assets_patch.py::test_patch_asset_invalid_sum_returns_422 --tb=long -p no:cacheprovider
```
Captured traceback:
```
>       assert response.status_code == 422
E       assert 404 == 422
```

Response body: `{"detail":"Not Found"}` — identical to 1.1.

## Diagnosis
Same root cause as 1.1: `_login_and_select` uses
`username="family"`, which doesn't exist in the seed
(`Italo`/`Ana` only). Login fails, no cookie, profile selection
fails, `require_active_profile` raises 404 before the handler
ever reads the asset.

The validator / 422 path is exercised by the test seed
(3 assets at 30/30/30, PATCH one to 50 → sum 110 → "Sobra 10%").
The validator's message is correct per `validators.py` and the
route re-emits it verbatim (see `assets.py:407-414`). The route
itself is correct.

## Decision
Per D2, **test edit** — same fix as 1.1 (replace `"family"` →
`"Italo"`). One change fixes both PATCH failures because they
share the same helper.

Verified: the validator's `"Sobra 10%"` message format matches
`assert body == {"detail": "Sobra 10%"}` in the test, so the 422
path will pass once the helper authenticates correctly.
