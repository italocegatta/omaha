## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem -> audit script | Only project templates/CSS are read; no DB or network access. |
| generated report -> user | Report stays in repo; not exposed via a public URL. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01-01 | Tampering | `inventory.render_page` / `cli.py` path resolution | mitigate | Resolve `--css`, `--templates-dir`, `--output` under repo root; reject `..` escapes after `Path.resolve()`. |
| T-01-01-02 | Information Disclosure | `reports/contrast_audit.html` | accept | Report is kept internal to the repo; no public route serves it in this phase. |
| T-01-01-03 | Denial of Service | Rendering templates with dummy contexts | mitigate | Use minimal primitive contexts; catch and report Jinja2 render errors without retry loops. |
</threat_model>
