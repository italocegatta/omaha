## Color

> **CORE OF THIS PHASE.** Defines the corrected token system that replaces the broken palette. All values are OKLCH unless noted. Contrast ratios verified via `color_resolver.contrast_ratio()` (coloraide, wcag21 method) against resolved `--bg = oklch(0.975 0.003 60)` and `--ink = oklch(0.20 0.01 60)`.

### 60/30/10 Split

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `--bg` | `oklch(0.975 0.003 60)` | Body background. Off-white, chroma ≈ 0. NOT cream. |
| Secondary (30%) | `--surface` | `oklch(1.0 0 0)` | Cards, modals, popovers. Pure white. |
| | `--surface-sunk` | `oklch(0.96 0.003 60)` | Form wells, input strips, table headers. Slightly recessed. |
| Accent (10%) | `--accent` | `oklch(0.42 0.09 150)` | Primary buttons, compare bar current fill, progress bar fill. Deep fern green, hue 150. |
| Destructive | `--negative` | `oklch(0.50 0.18 25)` | Loss text, delete confirm background, error indicators. |

### Accent Reserved For

| Use | Surface | Component | Rule |
|-----|---------|-----------|------|
| Primary button fill | `--accent` | `.btn-primary` | `background: var(--accent)` |
| Compare bar current fill | `--accent` | `.compare-bar-current-fill` | Per-asset progress |
| Progress bar fill | `--accent` | `.asset-progress-fill` | Per-asset progress |
| Per-class accent cycling | class tokens → `--class-color` | nth-of-type rules | Data visualization only |

Accent is explicitly **NOT** used for: text (use `--accent-ink` on accent backgrounds), borders, backgrounds for non-accent surfaces, error/success indicators, links (links use `--accent` for color in current CSS — Phase 3 fixes this).

### Tokens Unchanged (19 tokens)

All pass their real-world contrast pairs.

### Color Rules

1. All new and changed tokens use OKLCH. Hex values remain only for class colors 1-3 and 5 (already pass, deferred conversion) and `--negative-ink`/`--positive-ink` (white #ffffff is OKLCH-invariant).
2. Every foreground/background pair has an explicit token. No implicit pairs. No hardcoded `#fff` in component rules.
3. Legacy aliases (`--fg: var(--ink)`, `--muted: var(--ink-muted)`) are preserved indefinitely. Migration to canonical names is a future linting pass.
4. Semantic naming (`--ink`, not `--text-primary`). Value-named tokens (`--green-600`) are prohibited.
5. Surface-level tokens only. No component-scoped tokens (`--btn-primary-bg`) — use `--accent` + `--accent-ink`.
